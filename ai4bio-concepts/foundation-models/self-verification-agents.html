Core Concept

What is a Self-Verification Agent?

A self-verification agent is an LLM-based system that autonomously verifies its own outputs by querying external knowledge sources, rather than relying solely on the model's parametric knowledge or human validation.

Key Innovation: Instead of a human checking if the LLM's answer is correct, the agent automatically:

  1. Generates an initial answer
  2. Extracts verifiable claims from that answer
  3. Queries relevant databases for evidence
  4. Determines if each claim is supported, partially supported, or refuted
  5. Revises the answer based on verification results
Why This Matters: LLMs hallucinateâ€”they generate plausible but factually incorrect statements. In scientific domains like genomics, these hallucinations can mislead researchers. Self-verification reduces hallucinations from ~44% to ~19% in gene-set analysis.

Cascading Verification: Multi-Level Checking

Unlike single-pass verification, GeneAgent verifies at two levels of granularity:

  1. Coarse-grained: Verify the high-level "process name" (e.g., "RTK signaling pathway")
  2. Fine-grained: Verify detailed analytical narratives about individual genes

Key insight: Process names are verified twiceâ€”once directly, then again indirectly through the analytical narratives. This catches inconsistencies and prevents error propagation.

Example: Claim Verification

Claim: "ERBB2, ERBB4, FGFR2, FGFR4, HRAS, KRAS are involved in RTK signaling"

Agent Process:

  1. Extracts genes: ERBB2, ERBB4, FGFR2, FGFR4, HRAS, KRAS
  2. Queries g:Profiler and Enrichr for enrichment analysis
  3. Retrieves top enrichment: "MAPK signaling pathway"
  4. Decision: "Cannot be confirmed" (MAPK is related but not identical to RTK)
  5. Agent modifies claim to be more specific or removes it

Multi-Database Evidence Synthesis

Why Multiple Databases?

No single database has complete coverage of biological knowledge. GeneAgent integrates 18 databases via 4 Web APIs to achieve comprehensive verification:

Database Type Purpose Examples Use Case
GSEA Tools (8) Pathway enrichment GO, KEGG, Reactome, WikiPathways Verify process names
Gene-Centric (2) Individual gene functions GeneCards, NCBI Gene Verify gene-specific claims
Literature (1) Published research PubMed Central Find supporting evidence
Disease-Gene (7) Disease associations DisGeNET, OMIM, ClinVar Verify disease-gene links
Intelligent Database Selection: The agent doesn't blindly query all databases. It analyzes each claim to determine which databases are most relevant, improving efficiency and accuracy.

Applications Beyond Genomics

The self-verification paradigm can be applied to any domain with structured knowledge bases:

  • Drug Discovery: Verify drug-target interactions using ChEMBL, DrugBank
  • Clinical Medicine: Verify diagnoses using medical databases
  • Protein Biology: Verify protein functions using UniProt, PDB
  • Systems Biology: Verify pathway interactions using STRING, BioGRID

Implementation Best Practices

Design for Verification

Structure outputs to include explicit, verifiable claims rather than vague statements

Multi-Source Evidence

Use multiple databases to cross-validate findings and catch inconsistencies

Cascading Checks

Verify at multiple granularities to catch both high-level and detailed errors

Transparent Reporting

Always show which databases were queried and what evidence was found
